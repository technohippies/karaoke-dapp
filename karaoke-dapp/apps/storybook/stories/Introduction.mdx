# Karaoke dApp Design System

Welcome to the Karaoke dApp Storybook! This is where we document and showcase all UI components, patterns, and design decisions.

## Structure

Our stories are organized into the following categories:

### 🎨 Foundations
Core design tokens and primitives:
- Colors & Themes
- Typography
- Spacing & Layout
- Icons
- Animations

### 🧩 Components
Reusable UI components:
- Buttons
- Forms
- Cards
- Modals
- Navigation

### 🎤 Features
Domain-specific components:
- Karaoke Player
- Song Cards
- Voice Credits Display
- SRS Progress
- Wallet Connection

### 🔧 Patterns
Common UI patterns and compositions:
- Authentication flows
- Purchase flows
- Loading states
- Error boundaries

### 📚 Examples
Full page examples and integrations:
- Song Catalog Page
- Karaoke Session
- User Dashboard
- Settings

## Development Guidelines

1. **Accessibility First**: All components must be ARIA compliant
2. **RTL Support**: Components should work in both LTR (English, Chinese) and RTL (Uyghur) layouts
3. **Dark Mode**: All components should support light/dark themes
4. **Mobile First**: Design for mobile, enhance for desktop
5. **Localization**: Text should be externalized for i18n

## Tech Stack

- **React 19.1** with TypeScript
- **Tailwind CSS v4** for styling
- **Porto** for wallet integration
- **Vite 7** for fast builds
- **Storybook 9** for component documentation
